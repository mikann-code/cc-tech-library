## 概要

URL に動的な文字列を入れるとき、壊れないように安全な形式に変換するための関数 `encodeURIComponent` について。なぜ必要か、どんな場面で使うべきかを整理する。

## サンプルコード

### URL に使えない文字

URL には **使えない文字** がある:

| 文字 | 問題 |
| --- | --- |
| 空白 | URL は空白を許可しない |
| 日本語 渋谷 | ASCII 外の文字は使えない |
| `&` `=` `?` `#` | クエリの構造記号と衝突する |
| `/` `:` `@` | URL の予約文字 |

これらをそのまま URL に入れると **壊れる or 想定外の動作** をする。

### やっていること

```
"東京タワー 港区芝公園"
   ↓ encodeURIComponent
"%E6%9D%B1%E4%BA%AC%E3%82%BF%E3%83%AF%E3%83%BC%20%E6%B8%AF%E5%8C%BA..."
```

**危険な文字を `%XX` 形式に変換** する。受信側（サーバや Google マップ）が自動で元に戻す。

### 使うべき場面

| 場面 | 必要? |
| --- | --- |
| 動的な値を URL に埋め込む | 必須 |
| ユーザー入力を URL に含める | 必須（セキュリティ上も） |
| 日本語・空白・記号を URL に入れる | 必須 |
| 英数字だけを URL に入れる | 不要だが通しても害なし |

## ポイント

- URL に動的な文字列を入れるとき、壊れないように安全な形式に変換するための関数
- 空白、ASCII 外、`& = ? #`、`/ : @` などはそのまま URL に入れると壊れる or 想定外の動作になる
- 危険な文字を `%XX` 形式に変換し、受信側が自動で元に戻す
- 動的な値を URL に入れるなら、迷わず通す
- 英数字に対しては何もしないので、「常に通す」が一番安全
